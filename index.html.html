01 / Por que o Transformer?

O problema que precisava ser resolvido

Antes do Transformer, redes RNNs processavam texto palavra por palavra — esquecendo contexto distante e impossibilitando paralelização.

❌ RNN (antes de 2017)

  • Lê token por token, em sequência
  • Esquece contexto de palavras distantes
  • Impossível paralelizar no treinamento
  • "O gato… acordou" — perde o sujeito
→

✅ Transformer (2017+)

  • Processa todos os tokens em paralelo
  • Acessa qualquer posição diretamente
  • Treinamento muito mais rápido
  • Captura relações de longa distância
"Attention is All You Need" — Vaswani et al., Google Brain, 2017 · Um dos artigos mais citados da história da IA

03 / Embeddings & Positional Encoding

De palavras a vetores

Computadores só entendem números. Cada palavra é convertida num vetor numérico (embedding) e somada a um vetor de posição.

FRASE: "O gato comeu o peixe"

04 / Self-Attention

O coração do Transformer

Cada token olha para todos os outros e decide quanto atenção prestar a cada um.

↓ CLIQUE EM UMA PALAVRA PARA VER QUANTA ATENÇÃO ELA PRESTA ÀS OUTRAS

Selecione uma palavra acima

Attention(Q, K, V) = softmax( QKᵀ / √dₖ ) · V
Q

Query — Pergunta

O que este token busca? Representa a intenção de busca de cada token.
Ex: "Quem executou a ação?"

K

Key — Chave

O que este token tem a oferecer? O rótulo de cada token para ser encontrado.
Ex: "Sou o sujeito da frase"

V

Value — Valor

Qual é o conteúdo real? A informação que será ponderada pelos pesos de atenção.

07 / Saída do Decoder

Linear → Softmax → Probabilidades

Depois de todos os blocos Transformer, o vetor do último token é convertido numa distribuição de probabilidade sobre todo o vocabulário — e a palavra com maior probabilidade é gerada.

1. OUTPUT LOGIT

O vetor final do Decoder (512 dimensões) é multiplicado por uma matriz de pesos, produzindo 50.257 números — um para cada token do vocabulário. Esses são os logits: valores brutos ainda sem escala de probabilidade.

2. CAMADA LINEAR

A projeção linear é simplesmente y = xAᵀ + b. Os pesos desta camada são os mesmos da tabela de embeddings de entrada — um truque chamado weight tying que reduz o número de parâmetros do modelo.

3. SOFTMAX

Converte os logits brutos em probabilidades que somam 100%. A fórmula e^yᵢ / Σ e^yⱼ garante que tokens com logit maior recebam probabilidade muito maior — o efeito é amplificado pela exponencial.

TEMPERATURA — CONTROLANDO A ALEATORIEDADE

Antes do Softmax, os logits podem ser divididos por um valor T (Temperature):

Softmax( logits / T )
T < 1.0

Logits altos ficam proporcionalmente maiores. Modelo mais determinístico. Ideal para código e respostas factuais.

T = 1.0

Sem alteração. Distribuição original do modelo. Uso geral balanceado.

T > 1.0

Achata as diferenças. Tokens menos prováveis ganham chance. Modelo mais criativo. Ideal para escrita e brainstorming.

💡 Geração autoregressiva: o token gerado ("way") é acrescentado à sequência e o processo começa de novo — o modelo lê tudo novamente e gera o próximo token. É por isso que modelos como o ChatGPT e o Claude aparecem palavra por palavra: cada token depende de todos os anteriores.
08 / Sampling Strategy

Como o modelo escolhe o próximo token

Ter as probabilidades não basta — é preciso decidir como sortear o próximo token. A estratégia de amostragem controla o equilíbrio entre previsibilidade e criatividade.

PIPELINE COMPLETO DE GERAÇÃO
Logits ÷ T Temperature → Top-k / Top-p → Softmax → Sortear token

Greedy Search

Sempre escolhe o token com maior probabilidade. Determinístico — mesma entrada sempre gera a mesma saída.

✔ Consistente e rápido
✗ Texto repetitivo e previsível

Top-k

Mantém os k tokens mais prováveis, descarta o resto (−∞). O Softmax redistribui entre os k e sorteia um.

✔ Elimina tokens absurdos
✗ k fixo pode ser inadequado

Top-p (Nucleus)

Mantém os tokens cuja probabilidade acumulada ≥ p. O conjunto varia dinamicamente com a distribuição.

✔ Adapta-se à incerteza
✔ Mais natural e flexível

💡 Na prática: Claude, ChatGPT e Gemini usam Top-p + Temperature combinados. É por isso que a mesma pergunta pode gerar respostas diferentes a cada vez — e por que existe o parâmetro temperature na API.

O Transformer original tinha Encoder + Decoder. A comunidade criou variações especializadas — clique para explorar cada uma.
